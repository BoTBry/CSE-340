## Topic 1: Express Routing

In your project, routing is what decides what happens when someone visits a URL.

Here’s a simple example from a typical Express server:

app.get('/about', (req, res) => {
  res.render('about', {
    title: 'About Us',
    company: 'Service Network'
  });
});

This tells the server:

“When someone visits /about, show the about.ejs page and send it some data.”

Question:
In your own words, what do you think this line does?

app.get('/about', (req, res) => { ... })

User:
This gets the page which belongs to the "/about" route

Assistant:
Good, you’ve got the basic idea 👍

You’re close, but let’s tighten it a bit so it’s more accurate.

That line doesn’t directly fetch a page—instead, it creates a rule for the server.

It means:

“When a user visits /about, run this function and decide what to send back.”

So the server is the one responding, not just “getting a page.”

Here’s a more realistic example:

app.get('/products', (req, res) => {
  const products = ['Phone', 'Laptop', 'Tablet'];

  res.render('products', {
    title: 'Our Products',
    products
  });
});

Question:
What do you think the res.render() line is doing in this route?

User:
It looks for the EJS file then assigns the variable their property

Assistant:
Good thinking, Bryan — you’re on the right track 👍 but let’s refine it so it’s technically correct.

res.render() does two main things together:

1. It finds and loads the EJS file (like products.ejs)
2. It injects the data you pass in (like title and products) into that file
3. Then it converts it into HTML and sends it to the browser

Assistant:
Good choice—that’s a very common one to feel unsure about.

Express routing is how the server decides what to do when a user visits a certain URL.

Example:

app.get('/about', (req, res) => {
  res.send('About Page');
});

This means:
- app.get() listens for GET requests
- '/about' is the route path
- req is the request
- res is the response
